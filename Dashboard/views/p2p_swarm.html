Tabula Rasa — P2P Specialist Swarm

Distributed Discovery · Fisher-Verified · Serverless Exchange
Specialist (local/imported) Transfer / Export Integrity / Benchmark EWC Fisher Vault Exchange / Ledger DHT Agent

Phase 1 — LAN Discovery

  • mDNS broadcast on local subnet
  • Each node announces IP:port + specialist list
  • HTTP GET /specialists for available zips
  • Already works with current export/import API
  • Zero config, no DHT library needed

Phase 2 — DHT Registry

  • Install kademlia (pure Python, no deps)
  • Each node: specialist_name + sha256(fisher) + accuracy → DHT
  • Query by name + accuracy threshold
  • Fisher hash as content-addressable proof
  • Bootstrap from known peer list or mDNS

Phase 3 — Trust & Exchange

  • Piaget benchmark gate before import (90% gate)
  • Exchange ledger: you give one, you get one
  • RemoteSpecialist stub for live inference
  • Reputation: accuracy deltas over time
  • DHT-based NAT traversal for WAN peers

Implementation Roadmap

Week 1: mDNS discovery + LAN peer table in the Experiment Comparator. Nodes find each other on the same network automatically. Show peer specialists in the Exchange panel.
Week 2: RemoteSpecialist class — router can call a peer's model via TCP without downloading it. Useful for rare specialists you don't want to store locally.
Week 3: kademlia DHT overlay. Replace the HTTP registry with a distributed one. Fisher hash verification on both ends.
Week 4: Exchange protocol (give-to-get), reputation tracking, NAT punching for internet peers.
