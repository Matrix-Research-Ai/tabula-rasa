📖 Tabula Rasa AI Whitepaper v1.3 — June 2026

📜 Abstract

Tabula Rasa is a blank-slate artificial intelligence system that learns skills from scratch, one at a time, without pretraining on massive corpora. Unlike conventional LLMs that download a frozen brain from the internet, Tabula Rasa begins as a random weight matrix — a digital toddler knowing nothing — and develops “IQ” through structured training, self-improvement, and specialist specialization.

💭 1. Philosophy — The Blank Slate

Drawing from Piaget's developmental psychology and Locke's tabula rasa concept, the system starts with zero knowledge. Every capability — every concept, every operation, every word — must be learned through experience. This mirrors how biological intelligence develops: a human infant knows nothing and spends years building mental models through interaction.

🧪 V2 Input Format — Fused Carry-Digit Tokens

Digits are reversed (LSD first). Each column's output is a single fused token f'{carry}{digit}', e.g. "13" = carry=1, digit=3. No dots, no separators. 20 new vocab entries. 

Input:  38+45=
Output: 83+54=1308
        ^ LSD-first    ^ fused: carry=1,digit=3 | carry=0,digit=8
        Ones: 8+5=13 → token "13"
        Tens: 3+4+1=8 → token "08"

Fused tokens broke the 0.55 loss wall. Without them, carry and digit compete for gradient. With combined tokens, 100% of gradient goes to each answer position.

V1.2 Techniques (Proven)

Reversed digits — Digit strings reversed so carry propagates left-to-right. Eliminated the 5% accuracy cap that plagued the original format.

Loss masking — Prompt tokens set to ignore_index=-100. Forces model to focus 100% of capacity on answer generation. 2x convergence speed.

Windows DataLoader workaround — Fresh DataLoader created per epoch to avoid a PyTorch re-iteration hang on Windows. Manual batching disabled (too slow).

🧩 4. Scratchpad Reasoning

The central innovation of V1.3. Instead of engineering a Python pipeline of sub-transformers to handle carry propagation, the model outputs its internal reasoning as text tokens.

How It Works (V2 — Fused Tokens)

Expression: 38 + 45
Reversed:   83 + 54
Fused out:  1308
            ^-- ones: 8+5=13 → token "13"
                 ^-- tens: 3+4+1=8 → token "08"
Parsed: take every 2nd char: "38"
Accuracy: 51.3% vs 9% sub-transformer

Why Fused Tokens Beat Dots

Sub-transformer pipeline9% accuracy
V1 dot scratchpad~35%
V2 fused carry-digit51.3% (4L, 1M params)
The gradient problem: In V1 (dot format), carry and digit were separate tokens. The model could learn carry correctly but predict the wrong digit, getting 0 credit.

V2 fix: Fuse carry+digit into one token (20 new vocab entries). Now the model must predict both correctly in a single token. Loss dropped from stuck 0.55 to 0.01 in 2K steps.

Debugging Via Scratchpad

Every model output is human-readable. If the model generates .16.04 instead of .13.08, you know exactly which column miscalculated the carry. This is column-level debugging without writing any pipeline code.

🌐 7. P2P Specialist Swarm

A distributed network of Tabula Rasa nodes sharing trained specialists without a central server. Every node is a peer: training new specialists, seeding them to the network, and consuming specialists trained by others. Discovery happens via a Distributed Hash Table (DHT), transfers are direct TCP streams, and trust is enforced by cryptographic hashes and benchmark gates.

📡 Discovery (DHT)

Each node runs a Kademlia DHT agent. On boot, it announces: specialist_name + sha256(fisher.pt) + accuracy Query the DHT by name: "grammar" → list of peers serving grammar specialists with their accuracy scores and Fisher hashes. Bootstrap from mDNS-discovered LAN peers or a hardcoded seed list.

📥 Transfer

The existing zip format (model.pt + tokenizer.json + metadata.json) is the wire format. Peers stream it over HTTP or TCP. After download, two verification gates:

1. Fisher hash match — sha256 of received ewc_fisher.pt must match the DHT announcement. Reject on mismatch.
2. Piaget benchmark — run 100 held-out problems. Reject if below 90% accuracy.

🤖 RemoteSpecialist Stub

For rare or experimental specialists, a stub class forwards inference to the peer over TCP without downloading the weights. The router calls stub.generate(prompt) and doesn't know it's talking to a remote peer. This enables a marketplace of live models.

💱 Exchange Protocol

Optional give-to-get: "I'll give you my add(51%) if you give me your grammar(87%)." An exchange ledger tracks transactions. Prevents freeloading and incentivizes training high-quality specialists. Nodes that contribute rare specialists earn reputation.

Implementation Phases

Phase 1 LAN Discovery — mDNS multicast. Nodes find each other on the same subnet. Peer table in the Experiment Comparator. No DHT library needed.
Phase 2 DHT Overlay — kademlia (pure Python). Replace HTTP registry with distributed key-value store. Fisher hash verification on both ends.
Phase 3 Remote Stubs — RemoteSpecialist class for live inference. Router dispatches to remote peers transparently.
Phase 4 Exchange + NAT — give-to-get protocol, reputation tracking, NAT punching for internet-wide peers.

🏷️ 9. Conclusion

Tabula Rasa represents a return to first principles in AI research. Rather than scaling models to trillions of parameters and training on the entire internet, it asks: what can be learned from scratch, one skill at a time, by a system that is honest about its own ignorance?

The fused carry-digit approach proves that a 1M-parameter transformer can learn column-by-column carry propagation as text tokens — no Python pipeline needed. The Experiment Comparator turns every training run into a reproducible data point, and the Specialist Exchange lets the community share trained weights P2P. This is the "Tabula Rasa way": let the neural network learn the algorithm, don't engineer it — and share what you've built.
