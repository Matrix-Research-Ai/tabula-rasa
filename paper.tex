# Tabula Rasa: Learning Arithmetic from a Blank Slate

## Abstract

We show that a 1M-parameter transformer, trained from random initialization
with no pretraining, reaches **100% accuracy on 1-digit addition and subtraction**
in under 15,000 gradient steps (~8 minutes on a single GPU, <1 hour on CPU).
The same architecture reaches 31% on 1-digit multiplication using an
operation-specific partial-products scratchpad. These results establish a
reproducible lower bound for arithmetic capability in from-scratch
transformers and provide baselines for length generalization and curriculum
learning.

## Results

### Multi-Seed Accuracy

All experiments: 1M-parameter transformer (d=128, L=4, ff=512, heads=4),
trained from scratch with curriculum (1-digit, 15K steps), evaluated on
100 fresh problems. Five random seeds per operation.

| Operation | Mean | Std | Best | Worst | Converges at |
|-----------|------|-----|------|-------|--------------|
| Addition | **100.0%** | 0.0% | 100.0% | 100.0% | ~3,000 steps |
| Subtraction | **100.0%** | 0.0% | 100.0% | 100.0% | ~8,000 steps |
| Multiplication | 31.0% | 0.0% | 31.0% | 31.0% | ~12,000 steps |

### Key Findings

1. **From-scratch arithmetic is viable.** A 1M-parameter transformer
   achieves perfect accuracy on addition and subtraction with no pretraining,
   no curated data, and no task-specific architecture.

2. **Operation-specific scratchpads matter.** Addition uses fused
   carry-digit tokens (00-19). Subtraction uses the same format with a
   borrow marker chain. Multiplication uses partial-products decomposition
   with `<STEP>`/`<END>` markers. All three require different data
   formats matched to the algorithmic structure.

3. **Length generalization remains open.** All position encoding variants
   (ROPE, Abacus, None, Alibi) achieve 100% on 2-digit training but 0% on
   3-digit evaluation at 5K steps. Curriculum-based training efficiently
   learns 1-digit in ~3K steps but needs phase expansion for multi-digit
   generalization.

4. **Digit reversal and loss masking are critical.** Without digit reversal,
   multi-digit carry propagation fails (Causal-Carry Mismatch). Loss masking
   provides ~2x convergence speedup by preventing gradient waste on prompt
   tokens.

### Curriculum vs. Scratchpad Design

| Component | Addition | Subtraction | Multiplication |
|-----------|----------|-------------|----------------|
| Position encoding | ROPE | ROPE | ROPE |
| Digit reversal | Yes | Yes | No |
| Scratchpad format | `{carry}{digit}` fused | `{borrow}{digit}` fused | `<STEP>` partial products |
| Eval token budget | digits × 2 + 4 | digits × 2 + 4 | digits × 12 + 16 |

## Reproducibility

- **Code:** https://github.com/Matrix-Research-Ai/tabula-rasa
- **Install:** `pip install tabula-rasa-ai`
- **Train:** `tabula-rasa train add --quick`
- **Hardware:** NVIDIA RTX A4500 (L4), 24GB VRAM
- **CPU fallback:** 4-core, ~1 hour for 15K steps

All experiments use fixed seeds 0-4 with the `--seed` flag for exact
reproducibility. The full multi-seed pipeline is in
`experiments/run_multi_seed.py`.

## Conclusion

Tabula Rasa demonstrates that a small transformer can learn arithmetic from a
blank slate. The remaining challenges — multiplication convergence, length
generalization, and multi-digit capability — are well-defined architectural
problems rather than fundamental limitations.
